# Phase 3.5 — Resilience & Enrichment Design
**Date:** 2026-04-25
**Status:** Approved
**Phase:** 3.5

---

## 1. Goal

Make the dashboard work without hooks, enrich all agents with richer metadata (tokens, cost, model), and improve UX around status accuracy and hooks management.

---

## 2. Scope

**In scope:**
- Dual-mode ingestion: JSONL file watching as fallback + enrichment source
- Rich metadata extraction from JSONL: model, tokens, cost, cache stats
- Hooks become optional (dashboard works without them, hooks preferred for accuracy)
- Hooks management UI: install/remove BrowserWindows with description and example
- `uninstallHooks()` added to `installer.ts`
- Client UI: token gauge, model badge, teammate badge, waiting timer, stale dimming, 300ms debounce, dismissible banner

**Out of scope:**
- Terminating agents (UI-only stale handling, no process control)
- OS installer wizard pages (Windows NSIS/macOS DMG)
- Cloud relay, accounts (Phase 4)
- Writing to any `~/.claude/` files except `settings.json` (hooks install/remove only)

---

## 3. Architecture

Two ingestion paths feed into the existing WebSocket relay:

```
Hook path (unchanged, primary):
  Claude Code → POST :7777/api/events → ring buffer → WS broadcast → reducer

JSONL path (new, via chokidar):
  ~/.claude/projects/**/*.jsonl
    add event    → parse file → emit session_discovered (if unknown session)
    change event → read new lines →
        if agent hasHooks=false: synthesize status events (working/idle)
        always: extract enrichment data → emit enrich event
```

Each agent tracks `hasHooks: boolean` — false by default, flipped true on first real hook event. Status inference from JSONL runs **only** for `hasHooks=false` agents. Enrichment (tokens, cost, model) runs for **all** agents on every `change` event.

### Status Inference from JSONL (hookless agents only)

| JSONL signal | Inferred status |
|---|---|
| `tool_use` entry appears | `working` |
| `tool_result` after it | `idle` |
| `turn_duration` field present | `idle` |
| No new lines for 30min | stays as-is, UI dims (stale) |

### Enrichment Data (all agents)

Extracted from latest lines of each session's JSONL:

| Field | Source |
|---|---|
| `model` | `message.model` |
| `inputTokens` | `message.usage.input_tokens` (latest turn = current context fill) |
| `outputTokens` | `message.usage.output_tokens` (cumulative) |
| `cacheReadTokens` | `message.usage.cache_read_input_tokens` |
| `cacheWriteTokens` | `message.usage.cache_creation_input_tokens` |
| `costUSD` | `costUSD` (cumulative sum) |
| `turnDuration` | `turn_duration` (latest turn, ms) |
| `isSidechain` | `isSidechain` (bool) |

Context fill % = `inputTokens / modelContextWindow`. Model context window looked up from `model-limits.ts` config table. If model unknown, gauge shows raw token count only (no %).

### New WebSocket Message Types

```typescript
// New session found via JSONL (no hooks)
{ type: 'session_discovered', payload: AgentSnapshot }

// Enrichment update for any agent
{ type: 'enrich', sessionId: string, data: EnrichmentData }
```

## 5. Server Details

### `watcher.ts`

```typescript
import chokidar from 'chokidar'

// Maintains a per-file byte offset to read only new content on change events.
// Partial line buffer: hold incomplete lines until next change event.
// On add: parse full file, discover session.
// On change: read from last offset, process new complete lines only.
```

Watches `~/.claude/projects/**/*.jsonl`. Uses per-file byte offset + partial line buffer to handle lines mid-write (append-only files may be caught mid-write by chokidar).

### `model-limits.ts`

```typescript
export const MODEL_CONTEXT_WINDOWS: Record<string, number> = {
  'claude-opus-4-7':         200_000,
  'claude-sonnet-4-6':       200_000,
  'claude-haiku-4-5':        200_000,
  // add new models here as Anthropic releases them
}

export function getContextWindow(model: string): number | null {
  return MODEL_CONTEXT_WINDOWS[model] ?? null
}
```

Returns `null` for unknown models — client renders raw token count, no gauge %.

### `jsonl-status.ts`

Stateless function: takes an array of new JSONL lines, returns synthesized hook events (`PreToolUse`, `PostToolUse`, `Stop`) with the same shape as real hook events so the relay can emit them identically. The reducer sees no difference.

## 6. Desktop Details

### `installer.ts` — `uninstallHooks()`

Reads `~/.claude/settings.json`, removes only entries whose `command` matches the known curl command string. If a hook event's array becomes empty after removal, the key is deleted. Other hooks are untouched. Returns `boolean` (false if file unreadable/invalid JSON).

### Hooks Management Windows

Two small BrowserWindows (`hooks-setup.html`, `hooks-remove.html`) — self-contained HTML with inline styles, no framework. Each sends a message to main via `ipcRenderer` when the user clicks a button.

**`hooks-setup.html` content:**
- Heading: "Enable real-time monitoring"
- Explanation: hooks let Claude Code report status instantly; without them the dashboard polls JSONL files with ~500ms lag
- Shows the curl command being added
- States the file being modified: `~/.claude/settings.json`
- Buttons: "Install hooks" / "Skip for now"

**`hooks-remove.html` content:**
- Heading: "Remove hooks from Claude Code"
- Explanation: dashboard still works via JSONL fallback after removal
- Lists the 9 hook events being removed
- Buttons: "Remove hooks" / "Keep hooks"

### Tray Menu

```
The Office  (v1.2.3)
──────────────────────
Open Dashboard
──────────────────────
Install hooks          ← shown when hasHooks() = false
Remove hooks           ← shown when hasHooks() = true
──────────────────────
Quit
```

### First-Run Behavior

No blocking modal on first launch. The `HooksBanner` in the client dashboard surfaces the nudge instead. Tray menu always provides access to hooks setup.

## 7. Client Details

### New Types

```typescript
interface EnrichmentData {
  model: string
  inputTokens: number
  outputTokens: number
  cacheReadTokens: number
  cacheWriteTokens: number
  costUSD: number
  turnDurationMs: number
  isSidechain: boolean
}

interface AgentState {
  // ... existing fields ...
  hasHooks: boolean
  enrichment: EnrichmentData | null
  waitingSince: number | null   // Date.now() when status became 'waiting'
  lastActivityAt: number        // Date.now() of last update (any source)
}
```

### Reducer Changes

- `SESSION_DISCOVERED` — same as upsert in `INIT`, adds agent with `hasHooks: false`
- `ENRICH` — merges `EnrichmentData` into matching agent
- `PreToolUse` — set `waitingSince: null` (clear if was waiting)
- `Notification` — set `waitingSince: Date.now()`
- `PostToolUse` / `PostToolUseFailure` — **300ms debounce**: schedule transition to `idle`/`error`; if a new `PreToolUse` arrives within 300ms, cancel the scheduled transition

### `TokenGauge`

**On agent card (compact):** small progress bar (fill %) + short label `42k / 200k`. Grays out if model unknown (shows `42k tokens` only).

**In detail panel (expanded):**
```
Model:        claude-opus-4-7
Context:      ████████░░░░░░  42,310 / 200,000 tokens  (21%)
Output:       8,421 tokens generated
Cache:        12,000 read  /  3,200 written
Cost:         $0.087 this session   ← only shown when costUSD > 0
Last turn:    2.3s
```

Cost row is hidden when `costUSD` is 0 or absent — prepaid/subscription users see no cost display.

### `HooksBanner`

Dismissible top bar in the dashboard. Dismissed state stored in `localStorage`.

- **No hooks:** blue info bar — *"Install hooks for real-time updates (dashboard works without them)"* + "Install" button
- **Hooks active:** no banner (or minimal green "Hooks active" pill — TBD based on visual testing)
- **"Install" clicked:** opens `hooks-setup.html` BrowserWindow via IPC

### Teammate Badge

Shown on top-level agents (`parentSessionId` is null) where `isSidechain = false` and there are 2+ top-level agents in the same project group. Label: `teammate`. Subagents (`parentSessionId` set) remain unlabeled.

### Stale Dimming

Agent card opacity drops to 60% when `Date.now() - lastActivityAt > 30 * 60 * 1000` (30 minutes) and status is `idle` or `done`. No new status value. Tooltip on hover: *"No activity for 35 min"*.

### Waiting Timer

When status is `waiting`, show elapsed time next to the purple dot: *"waiting 2m 14s"*. Rendered via `Date.now() - waitingSince`, updated each second via `setInterval` in the component.

- [ ] Dashboard shows agents with no hooks installed (JSONL-only mode)
- [ ] Hook agents and hookless agents both display model, token count, cost in detail panel
- [ ] Token gauge shows correct fill % for known models, degrades to raw count for unknown
- [ ] `uninstallHooks()` removes only The Office entries, leaves other hooks intact
- [ ] hooks-setup and hooks-remove BrowserWindows open from tray and dashboard banner
- [ ] First launch shows no blocking modal; banner appears in dashboard instead
- [ ] Waiting timer counts up correctly and resets when agent resumes work
- [ ] Stale dimming applies after 30min of inactivity, not before
- [ ] 300ms debounce prevents working→idle flicker on fast tool calls
- [ ] Teammate badge appears on multiple top-level agents in same project, not on subagents
- [ ] All existing tests pass; new reducer actions have unit tests
